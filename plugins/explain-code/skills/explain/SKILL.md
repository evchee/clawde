# Explain Code

Explain the specified code or file in a clear, structured way. Target the explanation for someone who is familiar with programming but new to this codebase.

## Structure your explanation as:

### Purpose
What does this code do? What problem does it solve? One paragraph, no jargon.

### How It Works
Walk through the logic step by step. Use numbered steps for sequential flows, bullet points for parallel concepts.

### Key Components
List the main functions, classes, or modules and what each one is responsible for.

### Data Flow
Describe how data moves through the code: inputs, transformations, outputs.

### Dependencies
What external libraries, APIs, or other parts of the codebase does this depend on?

### Complexity
- **Time complexity**: Big-O analysis of the main operations
- **Space complexity**: Memory usage patterns
- **Cognitive complexity**: How hard is this to understand and maintain?

If `$ARGUMENTS` is provided, focus on explaining those specific files, functions, or concepts.
